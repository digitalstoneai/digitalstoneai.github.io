# PLAN.md

## Objective
Build a **static GitHub Pages portfolio website** for **DigitalStoneAI Studio / Jay** that presents a strong, professional, simple, clean, and powerful business landing page with supporting projects and links pages.

This plan is designed for execution by Codex or another AI-assisted development workflow. The goal is to move in phases, protect design quality, and avoid scope creep.

---

## Build Principles
Throughout every phase, keep these principles in mind:

- Professional over flashy
- Clear over clever
- Simple over bloated
- Credible over trendy
- Polished over overengineered
- Static-site friendly
- Easy to deploy on GitHub Pages
- Easy to update later with real case studies, logo, and media assets

---

### Goal

## Priority Order
If time or focus becomes limited, prioritize in this order:

1. Homepage
2. Shared design system
3. Projects page
4. One solid case study template
5. Links page
6. Additional placeholder case study pages
7. Optional gallery placeholder

This prevents the project from stalling on low-priority extras.

---

## Scope Control Rules
To prevent overbuilding, do not add:

- blog
- CMS
- backend forms
- complex animations everywhere
- unnecessary filters
- bloated skills grids
- large testimonial systems
- fake metrics
- too many pages
- features that do not support the core positioning

Keep v1 focused.

---

## Placeholder Content Strategy
Since final media and project details are not all available yet:

- use strong, professional placeholders
- keep claims realistic
- leave clear zones for future updates
- design sections so content replacement is easy
- do not hard-code assumptions that will cause redesign later
